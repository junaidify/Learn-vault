## рҹ”җ Authentication & Security

Complete stateless auth implementation with dual login support:

```
POST /api/auth/register      вҶ’ Register user (BCrypt password)
POST /api/auth/login         вҶ’ Login вҶ’ returns JWT access token
GET  /oauth2/authorization/google вҶ’ Initiate Google OAuth2
GET  /oauth2/callback/google      вҶ’ Oauth2SuccessHandler вҶ’ JWT issued
```

**Security Stack:**
- `SecurityFilterChain` with stateless session management
- `JwtFilter` вҶ’ validates token on every request before controller
- `CustomUserDetailsService` вҶ’ loads user from DB by email
- `DaoAuthenticationProvider` (Spring Security 6 constructor)
- `@PreAuthorize("hasRole('ADMIN')")` for role-based endpoint protection
- `GlobalExceptionHandler` via `@RestControllerAdvice` вҶ’ standardized error responses

## рҹ’і Payment Integration

Razorpay backend integration вҖ” fully functional order lifecycle:

```
POST /api/payments/create-order     вҶ’ Creates Razorpay order, returns orderId
POST /api/payments/verify           вҶ’ Verifies HMAC-SHA256 signature
POST /api/payments/webhook          вҶ’ Handles async payment events
```

**Flow:**
```
Client вҶ’ POST /create-order вҶ’ RazorpayClient.orders().create()
       вҶҗ { orderId, amount, currency, key }

Client pays via Razorpay checkout вҶ“

Client вҶ’ POST /verify вҶ’ HMAC(orderId + paymentId, secret)
       вҶ’ EnrollmentService.enroll(userId, courseId)
       вҶҗ { success: true }
```

## рҹ—„пёҸ Database Schema

Core JPA entities with proper relationship modeling:

```java
// UserEntity в”Җв”Җв”Җ @OneToMany в”Җв”Җв–ә EnrollmentEntity
// CourseEntity в”Җв”Җ @OneToMany в”Җв”Җв–ә EnrollmentEntity
// EnrollmentEntity has @ManyToOne to both User and Course

@Entity
public class EnrollmentEntity {
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id")
    private UserEntity user;

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "course_id")
    private CourseEntity course;

    @Enumerated(EnumType.STRING)
    private PaymentStatus paymentStatus;
}
```

**Tables:** `users` В· `courses` В· `enrollments` В· `payments`

## рҹҡҖ Getting Started

### Prerequisites

- Java 17+
- PostgreSQL running locally
- Razorpay test API keys (optional for payment testing)

### Setup

```bash
# Clone the repo
git clone https://github.com/junaidify/Learn-vault.git
cd Learn-vault

# Configure environment
cp src/main/resources/application.yml.example src/main/resources/application.yml
# Edit application.yml вҶ’ set DB credentials, JWT secret, Razorpay keys

# Run
./gradlew bootRun
```

### application.yml

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/learnvault
    username: your_db_user
    password: your_db_password
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true

app:
  jwt:
    secret: your_jwt_secret_here
    expiration: 86400000  # 24h in ms

razorpay:
  key-id: rzp_test_xxxx
  key-secret: your_razorpay_secret
```

## рҹ—әпёҸ What I Learned Building This

| Concept | Key Insight |
|--------|-------------|
| **Spring Security 6** | `DaoAuthenticationProvider` constructor changed вҖ” must call `setUserDetailsService()` explicitly |
| **JJWT 0.12.6** | `Jwts.parser().verifyWith(key)` replaces old `.setSigningKey()` API |
| **JPA Relationships** | Never use raw FK fields вҖ” use `@ManyToOne` with `@JoinColumn`, lazy fetch by default |
| **OAuth2 + JWT** | `Oauth2SuccessHandler` must generate JWT after OAuth2 success to stay stateless |
| **Exception Handling** | `@RestControllerAdvice` + typed `@ExceptionHandler` keeps error responses consistent |
| **DTO Pattern** | Service layer returns DTOs, never entities вҖ” decouples persistence from API contract |
